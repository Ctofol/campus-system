# 登录502错误排查指南

## 问题描述
登录时出现502错误（Bad Gateway），后端没有收到请求。

## 已确认的信息
- ✅ 后端服务正在运行（端口8000，进程ID: 13880）
- ✅ 后端可以通过curl访问（验证码接口测试成功）
- ✅ 前端配置的baseUrl正确（`http://192.168.0.216:8000`）
- ✅ 编译后的代码使用正确的IP地址
- ✅ 电脑IP地址: `192.168.0.216`
- ✅ manifest.json中urlCheck已设置为false
- ✅ 编译后的project.config.json配置正确

## 快速修复（推荐按顺序尝试）

### 🔥 方法1：使用清理脚本（最简单）
1. 双击运行 `fronted/clear_cache_and_rebuild.bat`
2. 在HBuilderX中重新运行到微信开发者工具
3. 在微信开发者工具中确认"不校验合法域名"已勾选

### 🔥 方法2：微信开发者工具设置（最常见原因）
这是最常见的问题！请仔细检查：
1. 打开微信开发者工具
2. 点击右上角"详情"按钮
3. 选择"本地设置"标签
4. **必须勾选**："不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"
5. 如果已经勾选，尝试：取消勾选 → 保存 → 重新勾选 → 保存

### 🔥 方法3：清除所有缓存
1. 在微信开发者工具中：工具 → 清除缓存 → 清除全部缓存
2. 关闭微信开发者工具
3. 在HBuilderX中停止运行
4. 删除 `fronted/unpackage` 文件夹
5. 重新运行到微信开发者工具

## 排查步骤

### 1. 检查微信开发者工具网络设置（最重要！）

在微信开发者工具中：
1. 点击右上角"详情"按钮
2. 在"本地设置"标签页中
3. **确保勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"**
4. 如果已经勾选，尝试取消勾选再重新勾选

### 2. 清除缓存并重新编译

#### 方法A：在微信开发者工具中
1. 点击菜单栏"工具" -> "清除缓存"
2. 选择"清除全部缓存"
3. 关闭微信开发者工具
4. 重新打开项目

#### 方法B：在HBuilderX中
1. 停止当前运行
2. 删除 `fronted/unpackage` 文件夹
3. 重新运行到微信开发者工具

### 3. 使用测试页面诊断

1. 在微信开发者工具中，手动输入路径访问测试页面：
   ```
   pages/test-connection/test
   ```

2. 点击"测试验证码接口"按钮
3. 查看返回结果

4. 打开微信开发者工具的"Console"标签，查看详细错误信息
5. 打开"Network"标签，查看实际发送的请求URL和响应

### 4. 检查防火墙设置

#### Windows防火墙
1. 打开"Windows Defender 防火墙"
2. 点击"允许应用通过防火墙"
3. 确保Python已被允许（专用网络和公用网络都勾选）

#### 或者临时关闭防火墙测试
```powershell
# 以管理员身份运行PowerShell
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled False
```

测试完成后记得重新开启：
```powershell
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True
```

### 5. 检查后端日志

在后端运行的终端中查看是否有请求日志。如果没有任何日志，说明请求没有到达后端。

### 6. 尝试使用真机调试

如果模拟器一直有问题，可以尝试：
1. 在微信开发者工具中点击"预览"
2. 使用手机微信扫码
3. 在真机上测试登录功能

注意：真机和电脑需要在同一个WiFi网络下。

### 7. 检查manifest.json配置

确认 `fronted/manifest.json` 中的小程序配置：
```json
"mp-weixin": {
  "appid": "wx0b00092e88476970",
  "setting": {
    "urlCheck": false  // 这个很重要！
  }
}
```

### 8. 手动测试网络请求

在微信开发者工具的Console中手动执行：
```javascript
wx.request({
  url: 'http://192.168.0.216:8000/common/captcha',
  method: 'GET',
  success: (res) => {
    console.log('Success:', res);
  },
  fail: (err) => {
    console.error('Fail:', err);
  }
})
```

查看返回结果，这可以帮助确定是代码问题还是网络问题。

## 常见原因和解决方案

### 原因1：未勾选"不校验合法域名"
**解决方案**：按照步骤1操作

### 原因2：编译缓存问题
**解决方案**：按照步骤2清除缓存

### 原因3：防火墙阻止
**解决方案**：按照步骤4配置防火墙

### 原因4：IP地址变化
如果电脑IP地址变化了，需要：
1. 查看当前IP：`ipconfig`
2. 更新 `fronted/utils/request.js` 中的IP地址
3. 重新编译

### 原因5：后端CORS配置问题
后端已经配置了允许所有来源，应该不是这个问题。但如果需要检查，查看 `backend/app/main.py` 中的CORS配置。
