# 校园运动健康系统 - 功能介绍文档

## 系统概述

校园运动健康系统是一个面向学校体育教学和学生健康管理的综合平台，支持学生端、教师端和管理端三种角色，提供运动记录、任务管理、课程学习、健康报备、跑团活动等核心功能。

**技术栈**:
- 前端: uni-app + Vue3
- 后端: FastAPI + Python
- 数据库: PostgreSQL / SQLite
- 部署: Docker

---

## 一、学生端功能

### 1.1 账号与认证

#### 注册登录
- **注册**: `POST /auth/register`
  - 支持手机号注册
  - 图形验证码验证
  - 自动生成访问令牌
  
- **登录**: `POST /auth/login`
  - 手机号 + 密码登录
  - 返回用户信息和班级信息
  
- **获取验证码**: `GET /common/captcha`
  - 生成4位随机验证码图片

#### 个人资料管理
- **查看个人资料**: `GET /users/profile`
  - 查看姓名、手机号、学号、班级、健康状态
  - 查看个性签名和头像
  
- **编辑个人资料**: `PUT /users/profile`
  - 修改姓名
  - 上传头像
  - 设置个性签名

---

### 1.2 运动记录

#### 运动数据提交
- **提交运动记录**: `POST /activity/finish`
  - 支持跑步（run）和体测（test）两种类型
  - 记录运动轨迹、距离、时长、配速
  - 支持视频上传（体测）
  - 自动AI评分（体测视频）
  - 打卡点记录

#### 运动历史查询
- **查看运动历史**: `GET /activity/history`
  - 分页查询个人运动记录
  - 显示运动类型、时间、距离、时长
  - 查看审核状态

#### 运动数据统计
- **今日数据摘要**: `GET /student/summary`
  - 今日里程、消耗卡路里、运动时长
  - 待办任务数量
  
- **总数据统计**: `GET /student/total-stats`
  - 总里程（km）
  - 总时长（分钟）
  - 总消耗（kcal）
  - 运动次数

---

### 1.3 任务管理

#### 任务查询
- **查看我的任务**: `GET /student/tasks`
  - 查看教师发布的运动任务
  - 显示任务要求（距离、时长、次数）
  - 显示任务状态（待完成、已完成、已过期）
  - 显示完成进度

#### 任务完成
- 通过提交符合要求的运动记录自动完成任务
- 系统自动判断是否达标

---

### 1.4 健康报备

#### 请假申请
- **提交健康报备**: `POST /student/health/request`
  - 支持请假（leave）和伤病（injury）两种类型
  - 填写申请原因
  - 上传附件（最多3张图片）
  - 支持上传请假证明、病历单等
  
- **查看申请记录**: `GET /student/health/requests`
  - 查看历史申请记录
  - 查看审批状态（待审批、已通过、已驳回）
  - 查看上传的附件

---

### 1.5 课程学习

#### 课程浏览
- **查看课程列表**: `GET /courses/`
  - 浏览所有公开课程
  - 按分类筛选（理论、技能、体能）
  - 查看课程封面、标题、描述

#### 课程学习
- **查看课程详情**: `GET /courses/{course_id}`
  - 查看课程内容列表
  - 查看是否已选课
  - 查看选课人数
  
- **选课**: `POST /courses/{course_id}/enroll`
  - 加入课程学习
  
- **退课**: `DELETE /courses/{course_id}/enroll`
  - 退出课程
  
- **查看我的课程**: `GET /courses/me/enrollments`
  - 查看已选课程列表

#### 学习进度
- **查看内容详情**: `GET /courses/content/{content_id}`
  - 查看视频、文档或链接内容
  
- **保存学习进度**: `POST /courses/content/{content_id}/progress`
  - 记录视频播放位置
  - 更新学习进度（0-100%）
  - 标记完成状态
  
- **查看学习进度**: `GET /courses/content/{content_id}/progress`
  - 查看当前学习进度
  - 获取上次播放位置

---

### 1.6 跑团联盟

#### 跑团管理
- **创建跑团**: `POST /run-group/create`
  - 设置跑团名称、描述、头像
  
- **加入跑团**: `POST /run-group/join`
  - 通过跑团ID加入
  
- **退出跑团**: `POST /run-group/leave`
  - 离开当前跑团
  
- **查看我的跑团**: `GET /run-group/my`
  - 查看跑团信息
  - 查看成员数、总里程、排名
  - 查看本月活动数
  
- **浏览跑团列表**: `GET /run-group/list`
  - 查看所有跑团
  - 分页浏览

#### 跑团活动
- **查看活动列表**: `GET /run-group/activity/list`
  - 查看所有跑团活动
  - 按状态筛选（报名中、进行中、已结束）
  
- **查看活动详情**: `GET /run-group/activity/{activity_id}`
  - 查看活动信息
  - 查看报名人数和名额
  
- **报名活动**: `POST /run-group/activity/apply`
  - 报名参加活动
  
- **取消报名**: `POST /run-group/activity/cancel`
  - 取消活动报名

#### 排行榜
- **查看跑团排行**: `GET /run-group/rank`
  - 按总里程排名
  - 查看各跑团成员数

---

### 1.7 打卡点

#### 打卡功能
- **查看打卡点**: `GET /checkpoints`
  - 查看所有打卡点位置
  - 查看打卡点半径范围
  
- **打卡**: `POST /activity/checkin`
  - 提交当前位置
  - 验证是否在打卡点范围内

### 2.5 审批管理

#### 健康报备审批
- **查看待审批报备**: `GET /teacher/health/requests`

  - 只显示本班学生的申请
  
- **审批报备**: `PUT /teacher/health/requests/{request_id}/review`

#### 运动记录审批
- **查看待审批记录**: `GET /approvals/pending`

  - 统一的审批入口
  
- **通过审批**: `POST /approvals/{approval_id}/approve`
  - 批准运动记录或健康报备
  
- **驳回审批**: `POST /approvals/{approval_id}/reject`
  - 驳回申请并说明原因

#### 运动记录审批
- **审批运动记录**: `POST /teacher/activity/{activity_id}/approve`
  - 审核学生提交的运动记录
  - 创建审核记录

## 五、数据模型

### 5.1 核心实体

#### 用户（User）
- 基本信息：姓名、手机号、角色
- 学生信息：学号、班级、分组
- 健康状态：正常/请假/受伤
- 个人资料：头像、个性签名

#### 班级（Class）
- 班级名称
- 班主任
- 学生列表

#### 运动记录（Activity）
- 运动类型：跑步/体测
- 来源：自由运动/任务
- 时间：开始时间、结束时间
- 状态：已完成/待审核/已通过

#### 运动数据（ActivityMetrics）
- 跑步数据：距离、时长、配速、轨迹
- 体测数据：次数、是否达标、视频、AI评分

#### 任务（Task）
- 任务信息：标题、类型、描述
- 要求：最小距离/时长/次数
- 截止时间
- 目标群体

#### 健康报备（HealthRequest）
- 类型：请假/伤病
- 原因
- 附件（图片列表）
- 状态：待审批/已通过/已驳回

#### 课程（Course）
- 课程信息：标题、描述、封面
- 分类：理论/技能/体能
- 是否公开

#### 跑团（RunGroup）
- 跑团信息：名称、描述、头像
- 统计数据：总里程、成员数、排名

## 七、API基础信息

### 7.1 接口规范

#### 基础URL
- 开发环境: `http://127.0.0.1:8000`
- 生产环境: `http://120.26.17.147:8000`

#### 认证方式
- Bearer Token认证
- Header: `Authorization: Bearer {token}`

#### 响应格式
```json
{
  "items": [],
  "total": 0,
  "page": 1,
  "size": 20
}
```

#### 错误响应
```json
{
  "detail": "错误信息"
}
```

### 7.2 状态码

- `200` - 成功
- `201` - 创建成功
- `400` - 请求参数错误
- `401` - 未认证
- `403` - 无权限
- `404` - 资源不存在
- `500` - 服务器错误

## 八、使用场景示例

### 8.1 学生完成跑步任务

1. 学生登录系统
2. 查看任务列表 `GET /student/tasks`
3. 开始跑步，记录轨迹
4. 提交运动记录 `POST /activity/finish`
5. 系统自动判断任务完成
6. 查看任务状态更新为"已完成"

### 8.2 教师发布并监控任务

1. 教师登录系统
2. 创建跑步任务 `POST /teacher/tasks`
   - 要求：3公里，30分钟内
   - 截止时间：本周五
3. 查看任务详情 `GET /teacher/tasks/{task_id}`
4. 实时查看学生完成情况
5. 任务截止后查看统计数据

### 8.3 学生请假流程

1. 学生提交健康报备 `POST /student/health/request`
   - 类型：请假
   - 原因：感冒发烧
   - 上传病历单图片
2. 教师查看待审批 `GET /teacher/health/requests`
3. 教师审批 `PUT /teacher/health/requests/{id}/review`
4. 学生健康状态自动更新为"请假"
5. 学生查看审批结果
