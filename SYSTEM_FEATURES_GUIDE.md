# 校园运动健康系统 - 功能技术指南 (v1.2.0)

## 📋 系统概述

校园运动健康系统（Campus Sports Health System）是一个跨平台的数字化体育管理方案。系统通过 **Uni-app** 实现多端覆盖，**FastAPI** 提供高性能异步后端支持。核心业务围绕学生运动监测、教师管理决策、校园体育社交三大支柱构建。

---

## 🛠 技术架构

| 模块 | 技术栈 | 状态 |
| :--- | :--- | :--- |
| **前端 (Frontend)** | Uni-app + Vue 3 + Composition API + Pinia | `[已完成]` |
| **地图服务 (Map)** | 腾讯地图 SDK (高精度 GPS + 轨迹纠偏) | `[已完成]` |
| **后端 (Backend)** | Python FastAPI + SQLAlchemy 2.0 + Pydantic | `[已完成]` |
| **数据库 (DB)** | SQLite (开发) / PostgreSQL (生产) | `[已完成]` |
| **认证 (Auth)** | JWT Token + OAuth2 Bearer | `[已完成]` |
| **部署 (DevOps)** | Docker + Nginx + Gunicorn/Uvicorn | `[已完成]` |

## 📅 开发计划表 (Milestones)

### 第一阶段：核心底座 (已达成 ✅)
- 后端基础架构模块化。
- 学生端基础跑步与记录逻辑实现。
- 基础认证与权限流程。

### 第二阶段：业务联动 (当前阶段 🔧)
- **数据同步优化**: 解决学生“阳光跑”详情与后台统计的毫秒级同步。
- **UI 鲁棒性**: 消除复杂网络环境下的白屏，增加全局 Loading 与 Error 边界。
- **Result 页面增强**: 重新设计运动结算页面的视觉表现力。

### 第三阶段：AI 与 大数据 (未来目标 🚀)
- **AI 视觉接入**: 实现完全自动化的 AI 体测（Mediapipe/OpenCV）。
- **智能预警系统**: 自动发现心率或步频异常的运动风险（结合可穿戴设备数据）。
- **多校区支持**: 扩展租户概念，支持一套系统多机构运行。
