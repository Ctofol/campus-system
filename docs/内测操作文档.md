# 学生端 App 使用操作文档

版本：v1.3  
适用对象：学生端 App 使用者、学生端内测参与者

## 1. 本次测试重点

本次学生端内测主要测试 3 个重点功能：

1. 任务跑：从“我的任务”进入老师发布的跑步任务，按任务要求完成跑步并提交。
2. 自由跑：不通过任务，直接进入跑步页面，自由开始、结束并提交运动记录。
3. 体测：进入 AI 体能测试，选择项目，录制或上传视频并提交。

附带体验内容：

- 首页是否正常显示。
- 历史记录是否能看到刚提交的跑步或体测。
- 课程、跑团、健康报备、个人资料等页面是否能正常打开和使用。

教师端和管理端不需要你操作。任务发布、健康报备审核、成绩查看等由负责人完成。

## 2. 安装和打开 App

### 2.1 安装 App

1. 打开负责人发给你的安装包下载链接，或接收负责人发送的安装包文件。
2. 点击下载。
3. 下载完成后，点击安装包。
4. 手机弹出安装确认时，点击“安装”。
5. 等待安装完成。
6. 安装完成后，桌面上会出现学生端 App 图标。
7. 点击 App 图标，打开应用。

Android 手机如果提示“不允许安装未知来源应用”：

1. 点击提示中的“设置”。
2. 找到当前浏览器、微信、文件管理器或安装器。
3. 打开“允许安装未知应用”。
4. 返回安装页面，重新点击安装。

iPhone 如果通过 TestFlight 测试：

1. 先安装 TestFlight。
2. 打开负责人发给你的 TestFlight 邀请链接。
3. 点击“接受”或“开始测试”。
4. 点击“安装”。
5. 安装完成后从桌面打开 App。

### 2.2 首次打开 App

1. 点击桌面上的 App 图标。
2. 等待进入登录页。
3. 如果出现权限弹窗，请按下面规则处理：
   - 位置权限：点击“允许”或“使用 App 时允许”。
   - 相机权限：点击“允许”。
   - 相册权限：点击“允许”。
   - 通知权限：建议点击“允许”。
4. 如果 App 打开后进入登录页，说明启动正常。

如果 App 闪退、一直停在启动页、白屏超过 10 秒，请截图或录屏反馈。

## 3. 登录账号和密码

### 3.1 登录规则

本次学生端测试使用：

- 账号：你的学号
- 密码：统一为 `123456`

示例：

| 学号 | 登录密码 |
| --- | --- |
| `2024001` | `123456` |

如果负责人单独发了测试账号，请以负责人发放的账号为准；密码仍优先尝试 `123456`。

### 3.2 登录步骤

1. 打开 App。
2. 进入登录页面。
3. 找到“账号”“手机号/学号”或类似输入框。
4. 输入你的学号。
5. 找到“密码”输入框。
6. 输入 `123456`。
7. 如果页面上有“学生端 / 教师端”切换，请点击或选择“学生端”。
8. 点击“登录”按钮。
9. 登录成功后，会进入学生首页。

登录成功的判断：

- 页面不再停留在登录页。
- 可以看到首页、运动、课程、我的等底部导航。
- 可以看到运动数据、任务入口或功能入口。

登录失败时请检查：

- 学号是否输错。
- 密码是否为 `123456`。
- 是否选中了学生端。
- 网络是否正常。

如果确认学号和密码正确但仍无法登录，请截图反馈错误提示。

### 3.3 退出登录

1. 点击底部“我的”。
2. 进入个人中心后，向下滑动页面。
3. 找到“退出登录”按钮。
4. 点击“退出登录”。
5. 如果弹出确认框，点击“确定”或“确认退出”。
6. 页面返回登录页，表示退出成功。

## 4. 先认识底部入口

登录后，页面底部通常有几个主要入口：

| 底部入口 | 作用 |
| --- | --- |
| 首页 | 看运动概览、快捷入口、任务提醒 |
| 运动 | 进入自由跑、体测等运动功能 |
| 课程 | 查看课程列表和课程详情 |
| 我的 | 查看任务、历史记录、健康报备、个人资料 |

如果你的安装包界面文字略有不同，请以页面实际显示为准。测试时只要找到相同功能入口即可。

## 5. 重点一：任务跑操作

任务跑是本次测试最重要的功能之一。请优先完成这一项。

### 5.1 进入我的任务

1. 登录 App。
2. 在首页查看是否有“我的任务”“任务提醒”“待完成任务”等卡片。
3. 点击“我的任务”或任务卡片。
4. 如果首页没有看到任务入口，点击底部“我的”。
5. 在“我的”页面中找到“我的任务”“任务列表”或“历史任务”入口。
6. 点击进入任务列表。

进入成功的判断：

- 能看到任务列表页面。
- 页面上有老师发布的任务。
- 任务卡片上一般会显示任务名称、任务类型、任务要求、截止时间、完成状态。

如果任务列表为空，但负责人说已经发布任务，请截图反馈。

### 5.2 查看任务详情

1. 在任务列表中点击一个“进行中”的任务。
2. 进入任务详情页。
3. 查看任务名称。
4. 查看任务类型，确认是跑步任务还是体测任务。
5. 查看任务要求，例如目标距离、目标时长、次数要求、截止时间。
6. 找到“去完成”“开始任务”“立即完成”或类似按钮。

重点检查：

- 任务要求是否看得懂。
- 距离、时长、截止时间是否显示完整。
- 按钮是否明显、能点击。

### 5.3 从任务进入跑步

1. 在跑步任务详情页点击“去完成”。
2. 页面应跳转到跑步页面。
3. 如果弹出位置权限，点击“允许”。
4. 等待地图定位到当前位置。
5. 查看页面上是否显示距离、时长、配速等数据。
6. 找到“开始”按钮。
7. 点击“开始”。

开始成功的判断：

- 计时开始增加。
- 按钮变成“暂停”“结束”或类似状态。
- 页面能看到当前位置或运动轨迹。

### 5.4 按任务要求完成跑步

1. 开始后，拿着手机行走或慢跑。
2. 尽量在室外或定位较好的位置测试。
3. 查看“时长”是否增加。
4. 查看“距离”是否增加。
5. 如果任务要求最低距离，请尽量跑到或走到对应距离。
6. 如果任务要求最低时长，请保持运动到对应时长。

注意：

- 如果只是短时间测试，可以按负责人要求跑较短距离。
- 真实测试时不要马上开始又马上结束，否则任务可能无法达标。
- 跑步过程中不要直接杀掉 App。

### 5.5 结束并提交任务跑

1. 达到任务要求后，点击“结束”。
2. 如果弹出确认框，点击“确认结束”。
3. 进入结算页后，查看本次距离、时长、配速。
4. 点击“提交”“保存记录”或“完成”。
5. 等待提交结果。
6. 提交成功后，返回“我的任务”。
7. 下拉刷新任务列表，或重新进入任务详情。
8. 查看任务状态是否变为“已完成”，或完成进度是否增加。

任务跑成功的判断：

- 页面提示提交成功。
- 历史记录中能看到本次运动。
- 任务详情中能看到本次运动记录。
- 任务状态变为已完成，或进度发生变化。

需要反馈的问题：

- 找不到任务入口。
- 任务列表为空。
- 任务详情打不开。
- 点击“去完成”没有反应。
- 从任务进入跑步后任务要求丢失。
- 运动提交成功，但任务状态没有更新。
- 历史记录有运动，但任务详情里没有。

## 6. 重点二：自由跑操作

自由跑是不通过老师任务，自己直接开始跑步。

### 6.1 进入自由跑

1. 点击底部“运动”。
2. 找到“跑步”“开始跑步”“自由跑”或类似按钮。
3. 点击进入跑步地图页面。
4. 如果弹出位置权限，点击“允许”。
5. 等待地图定位到当前位置。
6. 查看页面上是否有“开始”按钮。

### 6.2 开始自由跑

1. 点击“开始”按钮。
2. 看到计时开始变化，表示自由跑已开始。
3. 拿着手机行走或慢跑一段距离。
4. 查看“时长”是否增加。
5. 查看“距离”是否变化。
6. 查看“配速”是否有数值。
7. 查看地图上是否出现运动轨迹线。

如果你在室内，距离和轨迹可能不稳定。建议到室外测试。

### 6.3 暂停和继续

如果页面有“暂停”按钮：

1. 跑步开始后点击“暂停”。
2. 查看计时是否暂停。
3. 再点击“继续”或“恢复”。
4. 查看计时是否继续增加。

如果页面没有暂停按钮，可以跳过此项。

### 6.4 结束并提交自由跑

1. 跑步一段时间后，点击“结束”。
2. 如果弹出确认框，点击“确认结束”。
3. 查看结算页上的距离、时长、配速。
4. 点击“提交”“保存记录”或“完成”。
5. 等待提交完成。
6. 进入“我的”或“历史记录”，查看本次自由跑是否已保存。

自由跑成功的判断：

- 页面提示提交成功。
- 历史记录中能看到本次运动。
- 详情页能看到距离、时长和轨迹。

需要反馈的问题：

- 点击开始没有反应。
- 一直定位失败。
- 计时不动。
- 距离一直为 0。
- 轨迹不显示。
- 点击结束没有反应。
- 提交失败。
- 提交成功后历史记录没有本次运动。

## 7. 重点三：AI 体测操作

体测是本次测试重点之一。请重点测试能否进入、能否选择项目、能否录制或上传视频、能否提交成功。

### 7.1 进入体测页面

1. 在首页找到“AI 体能测试”“体测”或类似入口。
2. 如果首页没有看到，点击底部“运动”。
3. 在运动页面找到“AI 体测”“体能测试”或类似入口。
4. 点击进入体测页面。
5. 查看页面是否显示体测项目列表。

常见项目可能包括：

- 俯卧撑
- 仰卧起坐
- 跳绳
- 深蹲
- 其他负责人指定项目

### 7.2 选择体测项目

1. 在体测项目列表中选择负责人要求测试的项目。
2. 点击项目卡片。
3. 如果页面有“开始测试”按钮，点击“开始测试”。
4. 阅读页面上的测试说明。
5. 确认当前项目名称是否正确。

重点检查：

- 项目名称是否显示清楚。
- 项目说明是否能看懂。
- “开始测试”“上传视频”“选择视频”等按钮是否明显。

### 7.3 现场录制视频

如果负责人要求现场录制：

1. 点击“开始录制”“拍摄视频”或类似按钮。
2. 如果弹出相机权限，点击“允许”。
3. 手机打开相机后，把手机放在能拍到完整动作的位置。
4. 保持人物全身或上半身在画面中，不要离镜头太近。
5. 录制一段短视频。
6. 录制完成后，点击“使用视频”“完成”或“确定”。
7. 返回 App 后，点击“上传”“提交”或“开始分析”。
8. 上传时不要退出 App。
9. 等待页面提示上传成功或显示分析状态。

录制重点：

- 光线尽量充足。
- 动作人物尽量完整出现在画面里。
- 视频不要太长，按负责人要求录制即可。
- 录制完成后一定要点击上传或提交。

### 7.4 上传已有视频

如果负责人允许上传已有视频：

1. 点击“上传视频”“选择视频”或类似按钮。
2. 如果弹出相册权限，点击“允许”。
3. 从相册中选择一个 MP4 视频。
4. 点击“确定”或“完成”。
5. 回到 App 后，确认页面显示已选择视频。
6. 点击“上传”“提交”或“开始分析”。
7. 等待上传完成。

上传成功的判断：

- 页面提示“上传成功”。
- 页面显示“分析中”“待分析”“分析完成”或成绩结果。
- 历史记录或体测记录中能看到本次测试。

### 7.5 查看体测结果或状态

1. 上传完成后，留在当前页面查看状态。
2. 如果页面提示“分析中”，等待一会儿。
3. 如果有“刷新”“查看结果”按钮，点击查看。
4. 也可以进入“历史记录”，查看刚提交的体测记录。
5. 打开体测记录详情，查看成绩、次数、分析状态或失败原因。

需要反馈的问题：

- 点击体测入口没有反应。
- 体测项目列表为空。
- 选择项目后页面错误。
- 相机打不开。
- 相册打不开。
- 选择视频后无反应。
- 上传进度一直不动。
- 上传失败但没有原因。
- 上传成功后找不到记录。
- 分析状态一直不变化。

## 8. 附带内容一：首页和历史记录

### 8.1 首页

1. 点击底部“首页”。
2. 查看页面上方是否显示你的头像、昵称或欢迎语。
3. 查看是否显示今日运动、累计运动、任务提醒等信息。
4. 点击首页上的跑步、体测、任务入口，确认能跳转到对应页面。

需要反馈：

- 首页白屏。
- 首页一直加载。
- 文字重叠或看不清。
- 入口点击无反应。

### 8.2 历史记录

1. 点击底部“我的”。
2. 找到“历史记录”“历史任务”“运动记录”或类似入口。
3. 点击进入。
4. 查看刚提交的任务跑、自由跑或体测是否出现在列表中。
5. 点击一条记录进入详情。
6. 跑步详情重点看距离、时长、配速、轨迹。
7. 体测详情重点看项目名称、上传状态、分析状态或成绩。

需要反馈：

- 刚提交的记录没有出现。
- 详情打不开。
- 距离、时长、成绩明显不对。
- 地图轨迹不显示。

## 10. 附带内容三：健康报备和个人资料

### 10.1 健康报备

1. 点击底部“我的”。
2. 找到“健康报备”入口。
3. 点击进入健康报备页面。
4. 选择报备类型或原因。
5. 填写说明，例如“身体不适测试提交”。
6. 如果页面有时间选择，选择开始时间和结束时间。
7. 点击“上传附件”“添加图片”或类似按钮。
8. 从相册选择一张测试图片。
9. 图片显示在页面中后，点击“提交”。
10. 提交成功后，查看状态是否为“待审核”。

负责人审核后，可以再次进入健康报备页面查看状态是否变为“已通过”或“已驳回”。

### 10.2 个人资料

1. 点击底部“我的”。
2. 点击头像区域，或点击“编辑资料”。
3. 修改头像、昵称或个性签名。
4. 点击“保存”。
5. 返回“我的”页面，查看信息是否更新。
6. 重新进入编辑资料，确认内容是否保留。

需要反馈：

- 图片上传失败。
- 健康报备提交失败。
- 编辑资料保存失败。
- 保存后内容没有变化。

备注：
