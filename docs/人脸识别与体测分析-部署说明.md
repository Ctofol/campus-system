# 人脸识别与体测分析 — 部署说明

本文档说明翊晨运动（campus-system）中 **跑步起止人脸核验** 与 **体测视频分析** 的部署方式，便于内测、上线与运维交接。

相关代码：

| 模块 | 路径 |
|------|------|
| 人脸核验入口 | `backend/app/services/face_verify_service.py` |
| InsightFace 本地比对 | `backend/app/services/face_local_insightface.py` |
| 体测视频分析 | `backend/app/services/test_analysis_service.py` |
| 姿态计数 | `backend/app/services/pose_analyzer/analyzer.py` |
| 环境变量示例 | `backend/.env.example` |
| 数据库字段补丁 | `backend/app/db_migrate.py`（应用启动时自动执行） |

---

## 一、功能概览

### 1.1 跑步人脸核验

| 阶段 | 行为 |
|------|------|
| 小程序 | 开跑、结束各拍一张，上传至 `/upload/file`，提交时带 `start_face` / `end_face` 证据 |
| 后端 | 根据 `FACE_PROVIDER` 决定核验强度 |

| `FACE_PROVIDER` | 说明 | 是否计费 |
|-----------------|------|----------|
| `none`（默认） | 仅检查起跑、结束两张照片是否存在 | 否 |
| `local` | **InsightFace** 本地 1:1 比对（起止是否同一人） | 否（占用服务器 CPU/内存） |
| `tencent` | 腾讯云 IAI：活体 + 人脸比对 | 按次扣腾讯云费用 |

> **说明**：`local` 模式**不包含**云端级活体检测；仍依赖小程序现场摄像头拍照。要防翻拍可后续接反欺骗模型或改用 `tencent`。

### 1.2 体测视频分析

| 项目 | 说明 |
|------|------|
| 提交 | 学生上传视频 + `exercise_type`（`pull_up` / `sit_up` / `push_up`） |
| 后端 | 不再使用随机次数；提交后 `analysis_status=pending`，后台线程分析 |
| 算法 | OpenCV 运动启发式；若安装 `mediapipe` 可提升计数精度 |
| 查询 | `GET /activity/{id}/analysis-status`；重新分析 `POST /activity/{id}/reanalyze` |

## 二、环境变量（`backend/.env`）

复制示例并修改：

```bash
cp backend/.env.example backend/.env
```

### 2.1 人脸 — 推荐：本地 InsightFace（不开通腾讯云）

```env
FACE_PROVIDER=local
FACE_INSIGHTFACE_MODEL=buffalo_l
FACE_INSIGHTFACE_PROVIDERS=CPUExecutionProvider
FACE_LOCAL_MIN_SIMILARITY=0.42
FACE_BLOCK_ON_FAIL=true
```

| 变量 | 含义 | 建议值 |
|------|------|--------|
| `FACE_PROVIDER` | `none` / `local` / `tencent` | 内测推荐 `local` |
| `FACE_INSIGHTFACE_MODEL` | InsightFace 模型包 | `buffalo_l` |
| `FACE_INSIGHTFACE_PROVIDERS` | ONNX 执行器 | 无 GPU：`CPUExecutionProvider`；有 NVIDIA：`CUDAExecutionProvider` |
| `FACE_LOCAL_MIN_SIMILARITY` | 同人判定阈值（0~1） | 默认 `0.42`；误杀同一人可降到 `0.38`，漏检换人可升到 `0.48` |
| `FACE_BLOCK_ON_FAIL` | 人脸失败是否判跑步无效 | `true` |

### 2.2 人脸 — 可选：腾讯云

```env
FACE_PROVIDER=tencent
TENCENT_SECRET_ID=你的SecretId
TENCENT_SECRET_KEY=你的SecretKey
TENCENT_REGION=ap-guangzhou
FACE_MATCH_THRESHOLD=70
FACE_BLOCK_ON_FAIL=true
```

密钥在：[腾讯云控制台 → 访问管理 → API 密钥](https://console.cloud.tencent.com/cam/capi)  
需已开通 **人脸识别（IAI）** 中的「人脸静态活体检测」「人脸比对」。**不必购买资源包**，按量计费即可。

依赖：

```bash
pip install tencentcloud-sdk-python
```

TEST_MIN_COUNT_SIT_UP=10

## 三、依赖安装

在 `backend` 目录：

```bash
cd backend
python -m venv venv
# Windows: venv\Scripts\activate
# Linux:   source venv/bin/activate
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

`requirements.txt` 已包含（人脸本地 + 体测基础）：

- `insightface`、`onnxruntime`、`opencv-python-headless`

可选提升体测精度：

```bash
pip install mediapipe
```

可选腾讯云人脸：

```bash
pip install tencentcloud-sdk-python
```

## 四、InsightFace 首次使用

### 4.1 模型下载

首次启动或首次调用时，InsightFace 会从网络下载模型到：

```text
~/.insightface/models/
```

**无公网的服务器**：在一台有网机器执行一次测试，再将 `models` 目录打包上传到服务器相同路径，或打进 Docker 镜像。

### 4.2 本地命令行自测

```bash
cd backend
python scripts/test_insightface_local.py /path/to/start.jpg /path/to/end.jpg
```

输出示例：

```text
余弦相似度: 0.5812
展示分数(0-100): 58.1
通过
```

### 4.3 与业务的关系

- 照片必须能通过服务器本地路径读取：`/uploads/YYYYMM/xxx.jpg`（与 `upload.py` 存储一致）。
- 若路径为完整 `https://` URL 且文件不在本机磁盘，本地模式会跳过算法（`LOCAL_SKIP_NO_FILE`），仅当双照存在时通过——**部署时请保证上传落盘到 `backend/uploads`**。

## 五、部署方式

### 5.1 systemd（线上常用）

参考根目录 `DEPLOYMENT_GUIDE.md`。发版后：

```bash
cd /path/to/campus-system
git pull
cd backend && pip install -r requirements.txt   # 依赖变更时
# 编辑 backend/.env 或 systemd EnvironmentFile
sudo systemctl restart campus-backend
sudo journalctl -u campus-backend -n 100 --no-pager
```

**systemd 加载环境变量**：在 service 文件中增加，例如：

```ini
EnvironmentFile=/path/to/campus-system/backend/.env
```

### 5.2 Docker Compose

根目录 `docker-compose.yml` 中 `campus-backend` 建议增加：

```yaml
campus-backend:
  env_file:
    - ./backend/.env
  volumes:
    - ./backend/uploads:/app/uploads
```

发版：

```bash
docker compose up -d --build campus-backend
docker compose logs -f campus-backend
```

> 仅 `docker compose restart` **不会**更新镜像内 Python 依赖；改 `requirements.txt` 后必须 `--build`。

### 5.3 数据库字段

新字段（活动表、指标表）在应用启动时由 `db_migrate.py` 自动 `ALTER TABLE` 补充，无需单独跑 Alembic。若使用全新空库，`create_all` 也会建全表。

## 六、验证清单

### 6.1 人脸（`FACE_PROVIDER=local`）

- [ ] `pip install` 无报错，`import insightface` 成功
- [ ] `test_insightface_local.py` 同一人两张照相似度 ≥ 0.42
- [ ] 小程序完整跑一条：结算页 / 详情有 **人脸核验** 文案与 **相似度**
- [ ] 换人测试：应提示「非同一人」且 `face_verified=false`（`FACE_BLOCK_ON_FAIL=true` 时跑步无效）
- [ ] 教师端任务跑步详情：「起止人脸核验」展示分数

### 6.2 体测

- [ ] 提交视频后接口返回 `analysis_status: pending`
- [ ] 约 10~60 秒后 `GET /activity/{id}/analysis-status` 为 `success`，有 `count`、`qualified`
- [ ] 不再出现随机 8~15 次
- [ ] 任务体测带 `taskId` 时提交 `source=task`

### 6.3 日志排查

| 现象 | 可能原因 |
|------|----------|
| `SDK_NOT_INSTALLED` | 未装 `tencentcloud-sdk-python` 却设了 `tencent` |
| `InsightFace init failed` | 模型未下载完、磁盘满、缺 `onnxruntime` |
| `未检测到人脸` | 侧脸、过暗、人脸过小 |
| `LOCAL_SKIP_NO_FILE` | 图片 URL 非本机 `uploads` 路径 |
| 体测一直 `pending` | 后台线程异常，查 `campus-backend` 日志；视频路径不存在 |

## 七、阈值调优建议

收集内测样本（同一人 / 换人 / 弱光各 10 组），记录 `face_match_score`，再调整：

| 目标 | 调整 |
|------|------|
| 减少同一人误杀 | 降低 `FACE_LOCAL_MIN_SIMILARITY`（如 0.38） |
| 减少换人漏检 | 提高（如 0.48~0.50） |

体测次数阈值修改 `TEST_MIN_COUNT_*` 或任务里的 `min_count`。

## 八、安全与合规

- **不要**将 `TENCENT_SECRET_KEY` 或任何 `.env` 提交到 Git
- API 密钥仅授予必要子账号权限（若用腾讯云）
- 隐私政策中说明：起止人脸仅用于本次跑步核验与申诉复核
- 照片保留周期按学校要求配置（可在运维层定期清理 `uploads` 旧文件）

## 九、常见问题

**Q：不想用腾讯云，最低配置是什么？**  
A：`FACE_PROVIDER=local` + 安装 `insightface` + `onnxruntime`，无需 SecretId。

**Q：能否只用「有两张照片」？**  
A：可以，设 `FACE_PROVIDER=none`。

**Q：InsightFace 和腾讯云能同时开吗？**  
A：环境变量只能选一个 `FACE_PROVIDER`；要切换需改配置并重启后端。

**Q：体测可以不装 mediapipe 吗？**  
A：可以，使用 OpenCV 运动启发式计数，精度略低。

**Q：官方文档在哪？**  
A：[deepinsight/insightface](https://github.com/deepinsight/insightface)
