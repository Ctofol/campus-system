## 二、本次提交（431c5e8）更新摘要

面向用户反馈与上线前体验，完成以下前端改动（**无新增后端接口依赖**，跑团发布活动等为纯前端优化）。

### 1. 跑步模块

| 项 | 内容 |
|----|------|
| 地图「我的位置」 | 加大自定义定位图标；定位成功后常显「我的位置」气泡；配合系统蓝点 |
| 轨迹展示 | 跑步轨迹加粗、白边、方向箭头；左下角图例（我的位置 / 轨迹 / 起点） |
| 起终点 | 开始跑步标记「起」，结束标记「终」 |
| 首页乱码 | 修复「查看全部 &gt;」显示问题，改为 CSS 箭头 |
| 计步/定位 | 延续此前版本：`aab32df` / `8fc17e7` 的真机计步、GPS 兜底、隐私监听等（本提交以地图与文案为主） |

### 2. 跑团 · 发布活动（用户反馈）

| 反馈 | 处理 |
|------|------|
| 选「目标距离」时键盘挡住输入框 | 表单改为 `scroll-view`，监听键盘高度，底部按钮随键盘上移 |
| 活动时间两个框不直观 | 合并为**一个**日期时间选择框（`multiSelector`） |
| 顶部两个「发布活动」标题 | 使用自定义导航栏（`navigationStyle: custom`） |

涉及文件：`create-activity.vue`、`activity-create.vue`、`activity-datetime-picker.js`、`pages.json`。

### 3. 个人资料与头像

| 项 | 内容 |
|----|------|
| 换头像方式 | 改为**相册/拍照**选图（`avatar-picker.js`），与微信公众平台「选中的照片/摄像头」隐私项一致 |
| 资料页布局 | 头像区避开微信胶囊按钮；去掉冗长隐私说明挤占一行 |
| 教师资料 | `teacher/profile/edit.vue` 与学生端同步 |

### 4. 登录与请求

| 项 | 内容 |
|----|------|
| 401 过期 | 统一 `handleSessionExpired`，避免多接口重复弹窗 |
| 未登录 | 无 token 不发起业务请求，引导登录 |
| 媒体地址 | `resolveMediaUrl` 拼接静态资源 |
| 启动校验 | `App.vue`、`tab/home.vue` 检查 token |

### 5. 其他

- `sunshine/detail.vue`：页面整理与文案修正（含「40 次以上」等显示）
- `manifest.json`：清理无效定位 permission 配置项
- `student-home.vue`：教师首页「全部」链接箭头样式统一

## 四、部署与运营建议

| 步骤 | 说明 |
|------|------|
| 1. 拉取代码 | `git fetch && git checkout codex/login-account-compat && git pull` |
| 2. 编译上传 | HBuilderX 发行微信小程序，版本号递增 |
| 3. 隐私指引 | 勾选**位置**、**选中的照片/视频**、**摄像头**等；头像勿勾「微信昵称头像」若仅用相册 |
| 4. 提审发布 | 隐私指引审核通过后再让用户体验换头像、定位 |
| 后端 | 若仅用本次 UI/鉴权/相册头像：**可不更后端**；若使用跑团编辑资料接口，需部署含 `44c5280` 的后端 |

## 五、已知限制与后续建议

1. **部分用户定位仍失败**：与隐私审核、系统权限、室内 GPS 有关；需用户更新小程序并授权。  
2. **步数偶发为 0**：传感器权限或「只晃手机不走」；已有里程估算兜底，需真机走路验证。  
3. **建议跟进**：传感器启动失败时在页面常驻提示；微信端首次定位增加周期性重试。
